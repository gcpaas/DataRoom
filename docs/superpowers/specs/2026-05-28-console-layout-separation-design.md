# 控制台左右区域分离布局设计

## 背景

当前控制台页的目标是：

- 左侧显示一个精简菜单卡片
- 右侧显示 `user / profile / log` 路由页面
- 两边视觉上必须是两个独立区域

这次需求澄清后，用户明确否定“左右仍然挂在同一个控制台外层容器下”的做法，并确认右侧需要保留各个页面自己原本的白底、边框和容器样式。

## 已确认需求

### 路由与功能

- 控制台路由结构保持现状
- `/dataRoom/console/log` 继续作为访问日志页面入口
- 左侧菜单保留：
  - 用户管理
  - 个人信息
  - 访问日志

### 布局关系

- `dr-console-layout` 只负责左右排布
- `dr-console-layout` 本身应当透明
- `dr-console-layout` 不承担背景、边框、圆角、整体容器感
- 左右两边之间保留 `16px` 间距

### 左侧区域

- 左侧必须是独立白色卡片
- 左侧卡片距离顶部、左侧、底部均为 `16px`
- 左侧卡片保留标题“控制台”
- 左侧卡片内保留精简菜单和当前激活样式

### 右侧区域

- 右侧不再额外包统一白底卡片
- 右侧只作为透明的路由内容承载区
- 右侧继续保留各页面自己原本的白底边框样式
- 不允许因为控制台壳层再次制造“右侧总卡片”

## 设计方案

### 结构调整

`ConsoleLayout.vue` 最终只保留三个职责明确的层次：

1. `dr-console-layout`
   - 纯布局容器
   - 只负责 `flex` 横向排布和中间间距

2. `dr-console-layout__sidebar`
   - 左侧独立白色菜单卡片
   - 负责标题、菜单项、边框、圆角、内部留白

3. `dr-console-layout__content`
   - 右侧透明路由承载区
   - 只承载 `RouterView`
   - 不提供统一白底、边框、圆角

### 要删除或降级的壳层职责

以下做法不再保留：

- 让 `dr-console-layout` 带统一背景，形成整体控制台容器
- 为右侧内容再包一层统一白底边框卡片
- 使用 `sidebar-pane / content-pane` 这类让左右继续从属于同一视觉壳层的结构

### 视觉结果

最终页面应呈现为：

- 左边是一张独立菜单卡片
- 右边是当前业务页面本身
- 两边只是并排，不属于一个整体面板
- 中间不使用竖线分割，而是依靠间距和各自边框自然分开

## 实现约束

- 仅调整 `dataRoomFront/src/dataroom-packages/console/ConsoleLayout.vue`
- 不改动控制台子路由逻辑
- 不修改 `user / profile / operationLog` 三个页面内部业务逻辑
- 样式必须继续遵循 `docs/design/DESIGN.md`
  - 使用 Element Plus 变量
  - 不引入硬编码颜色
  - 不覆盖 Element Plus 内部样式

## 风险与处理

### 风险 1：右侧页面失去统一外层后边距不一致

处理方式：

- 接受右侧页面按各自现状展示
- 如果个别页面暴露出贴边问题，后续单独在该页面内部修正，而不是回到控制台壳层补统一卡片

### 风险 2：控制台壳层仍残留整体背景观感

处理方式：

- `dr-console-layout` 保持透明
- 不给它设置整体背景块、边框、圆角、包裹留白

## 验证

- `ConsoleLayout` 模板结构符合“左卡片 + 右透明承载区”
- 左侧卡片距离顶部、左侧、底部为 `16px`
- 左右之间间距为 `16px`
- 右侧不再出现统一外层卡片
- `npm run type-check`
- `npx eslint src/dataroom-packages/console/ConsoleLayout.vue`
