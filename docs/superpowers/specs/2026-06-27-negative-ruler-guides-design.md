# 大屏设计器标尺负坐标与参考线设计

## 背景

视觉大屏设计器已经支持画布平移、缩放、标尺和参考线，但当前标尺刻度与参考线位置都被限制在画布坐标 `0..宽高` 范围内。用户把视口平移到画布左侧或上方时，标尺不会显示负坐标；从标尺拖出的参考线也不能放在画布外的负坐标区域。

本次改动目标是让标尺和用户自定义参考线与当前可见工作区一致：只要用户能看到画布外的负坐标区域，就能看到对应负刻度，也能创建、拖动和编辑负坐标参考线。

## 目标

- 横向与纵向标尺按当前视口显示完整坐标范围，包括负数、0 和正数。
- 用户可以在可见负坐标区域创建、拖动、保存和重新加载参考线。
- 属性面板中的参考线位置输入允许负值，并保留有限数值清洗。
- 用户自定义负参考线参与 Moveable 吸附。
- 内置 50px 间隔吸附线仍只生成在画布内部，避免自动生成大量画布外吸附线。
- 保持现有页面配置结构，不新增后端接口，不改变页面生命周期。

## 非目标

- 不改变组件坐标模型，不限制或扩展组件本身能否被拖到画布外。
- 不新增无限画布能力。
- 不新增参考线命名、分组、颜色或批量管理能力。
- 不改变 `PageDesigner` 网格设计器。
- 不调整标尺视觉风格，继续遵循 `docs/design/DESIGN.md` 中的画布标尺规范。

## 推荐方案

采用“当前可见视口边界”作为负坐标参考线的自然边界。

标尺刻度根据视口左上和右下对应的画布坐标计算，不再把起点强制夹到 `0`。当视口展示画布左侧或上方空间时，刻度会显示 `-100`、`-50`、`0`、`50` 等值；当视口只展示画布内部时，行为与当前基本一致。

参考线位置允许小于 `0`，但通过标尺创建和拖动时不允许落到当前可见工作区之外。这样既支持负坐标，又避免拖拽交互产生不可见参考线。通过属性面板输入的值会做有限数值清洗，但不会在加载或普通编辑时把已有离屏参考线强行夹回当前视口。

## 受影响模块

### `dataRoomFront/src/dataRoom/visual-screen-designer/ruler.ts`

- 更新 `getRulerTicks()`：
  - 使用当前视口起点和终点对应的画布坐标计算 tick 范围。
  - 移除 `Math.max(0, ...)` 和跳过负值的逻辑。
  - 保留现有步长、主刻度和 label 间距策略。
- 调整参考线位置清洗：
  - 保留有限数值检查和四舍五入。
  - 不再默认把负值夹到 `0`。
  - 增加按当前视口计算参考线边界的 helper，供拖拽和创建参考线复用。
- 保持 `getVisualScreenMoveableGuidelines()` 的内置吸附线生成范围为画布内部，同时继续合并用户自定义参考线。

### `dataRoomFront/src/dataRoom/visual-screen-designer/RulerOverlay.vue`

- 创建参考线时，根据指针所在位置换算画布坐标，允许负数。
- 拖动参考线时按当前可见工作区边界限制位置。
- 创建参考线的命中判断从“必须在画布 `0..宽高` 内”调整为“穿过对应标尺后，落在当前可见工作区内”。
- 预览参考线使用同一套坐标计算，确保拖拽反馈与最终落点一致。

### `dataRoomFront/src/dataRoom/visual-screen-designer/ControlPanel.vue`

- 参考线位置输入允许负值。
- 更新输入值时复用有限数值清洗逻辑，避免 `NaN`、`Infinity` 进入配置。
- 继续使用 Element Plus 默认表单组件样式，不覆盖 `.el-*` 内部样式。

## 数据与兼容性

页面配置仍使用现有结构：

```ts
interface VisualScreenGuide {
  id: string
  position: number
  locked: boolean
}
```

已有页面中 `position >= 0` 的参考线保持不变。新版本保存的负坐标参考线会以同一字段持久化。旧版本如果打开这些配置，可能仍会按旧逻辑把负数归零；本次只保证新版本自身的保存与加载一致。

## 交互细节

- 横向标尺显示 x 坐标，纵向标尺显示 y 坐标。
- 负数 label 使用普通负号文本，例如 `-50`。
- 主刻度和次刻度密度仍由当前缩放比例决定。
- 锁定参考线仍不能拖动或删除，负坐标不改变锁定语义。
- 全局“锁定参考线”仍会禁止创建和拖动。
- 双击删除参考线的行为不变。
- 如果参考线位于当前视口之外，它不会显示，但配置仍可在属性面板中保留和编辑。

## 错误处理与边界

- 加载和属性面板编辑时，非有限位置值统一回退到 `0`。
- 标尺拖拽和创建时，非有限位置值回退到当前可见范围内的安全值。
- 画布宽高异常时，不影响负刻度生成；参考线边界以视口换算结果为准。
- 缩放比例异常时沿用现有安全缩放兜底。
- 不引入后端异常处理，不涉及 Java `catch` 块。

## 验证计划

- 运行 `npm run type-check`。
- 如修改样式，额外运行 `npm run lint`。
- 补充或更新前端纯函数测试时，覆盖以下场景：
  - 视口展示负坐标区域时，`getRulerTicks()` 返回负数、0 和正数刻度。
  - 负坐标参考线在归一化后被保留。
  - 用户自定义负参考线被加入 Moveable 吸附线。
  - 非有限参考线位置被清洗，不进入最终配置。
- 手动验证：
  - 平移画布到左侧或上方，标尺显示负刻度。
  - 从标尺拖出参考线到负坐标区域，参考线可见并可拖动。
  - 在属性面板输入负坐标，保存后重新打开仍保留。
  - 内置画布吸附线仍只在画布内部自动出现。
