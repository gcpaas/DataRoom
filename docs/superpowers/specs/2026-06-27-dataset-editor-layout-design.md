# 数据集编辑页布局优化设计

## 背景

当前数据集编辑弹窗最多分为三块：表结构、基础设置、数据预览。SQL 和 Excel 数据集会显示表结构，字段列表则嵌在基础设置内；其他类型多数是基础设置与数据预览左右排布。

本次优化将编辑体验统一为工作台式布局：

- SQL、Excel 等能读取元数据的数据集显示左侧表结构。
- JSON、HTTP、ES、WebSocket、MQTT 不显示左侧表结构。
- 右侧统一为上下两块：上方基础设置、下方数据预览区域。
- 右侧上下两块都是折叠面板，点击标题栏可折叠，默认全部展开。
- 数据预览区域内分为两个 Tab：数据、字段。

## 目标

1. 统一所有数据集类型的编辑弹窗布局。
2. 基础设置使用一行两列展示普通表单项。
3. 大块编辑区、表格类配置和入参配置独占整行。
4. 字段列表从基础设置中移到右下的“字段”Tab。
5. 数据预览和字段列表的现有内容、行为和数据更新逻辑保持一致。
6. 样式遵循 `docs/design/DESIGN.md`，只使用 Element Plus 变量和默认组件样式。

## 推荐方案

采用“统一布局骨架 + 各类型声明式分区”的方案。

改造 `DatasetEditorLayout.vue`，让它负责统一结构：

- 可选左侧 `metadata` 区域。
- 右上 `settings` 折叠面板。
- 右下 `preview` 折叠面板。
- 预览面板内置“数据 / 字段”Tab。

各数据集编辑器继续维护自身业务逻辑、校验、测试和字段解析逻辑，只调整模板插槽和排版归属。不重构成完全配置驱动表单，避免扩大改动范围。

## 布局结构

### 有表结构的数据集

适用于 SQL、Excel。

```text
┌──────────────┬──────────────────────────────┐
│ 表结构        │ 基础设置（可折叠，默认展开）   │
│              ├──────────────────────────────┤
│              │ 数据预览（可折叠，默认展开）   │
│              │ ┌──── 数据 ────┬──── 字段 ───┐ │
│              │ │ 当前预览表格  │ 字段列表表格 │ │
└──────────────┴──────────────────────────────┘
```

左侧表结构保留当前能力：刷新、折叠、懒加载表字段、点击节点复制名称。

### 无表结构的数据集

适用于 JSON、HTTP、ES、WebSocket、MQTT。

```text
┌─────────────────────────────────────────────┐
│ 基础设置（可折叠，默认展开）                  │
├─────────────────────────────────────────────┤
│ 数据预览（可折叠，默认展开）                  │
│ ┌──── 数据 ────┬──── 字段 ───┐               │
│ │ 当前预览表格  │ 字段列表表格 │               │
└─────────────────────────────────────────────┘
```

没有表结构时，右侧区域横向占满弹窗。

## 折叠面板行为

- “基础设置”和“数据预览”都是面板标题栏可点击折叠。
- 默认两个面板都展开。
- 折叠基础设置时，数据预览面板获得更多纵向空间。
- 折叠数据预览时，基础设置面板获得更多纵向空间。
- 两个面板都允许折叠，但不清空数据、不触发保存、不重置 Tab。
- 折叠状态只作为当前弹窗会话 UI 状态，不要求持久化。

## 数据预览 Tab

右下数据预览区域包含两个 Tab：

- `数据`：显示当前 `DatasetPreviewTable` 内容，保持现有效果。
- `字段`：显示当前各编辑器内的字段列表表格，字段名、类型、描述和空状态保持现有效果。

字段列表的生成和更新逻辑保持不变：

- 测试数据集后自动更新字段列表。
- JSON 字段解析后更新字段列表。
- WebSocket 测试样本处理后更新字段列表。
- MQTT 测试或字段映射处理后更新字段列表。
- 已配置的字段类型、描述尽量保留当前逻辑。

## 基础设置排版规则

普通字段一行两列。

以下类型的配置独占整行：

- SQL语句
- JSON数据
- 请求体
- 查询报文
- 消息处理脚本
- 测试样本
- 字段映射表
- 入参配置类
- 请求头表格

表单使用 Element Plus 默认表单组件样式。业务样式只控制外层 grid、间距、滚动和面板容器，不覆盖 `.el-*` 内部样式。

## 各数据集基础设置方案

| 数据集 | 左侧表结构 | 两列普通字段 | 独占整行字段 | 右下 Tab |
|---|---|---|---|---|
| SQL | 显示表/字段树，可折叠、刷新、点选复制 | 数据集名称、数据源 | SQL语句、入参配置 | 数据 / 字段 |
| Excel | 显示 Excel 表/字段树，可折叠、刷新、点选复制 | 数据集名称 | SQL语句、入参配置 | 数据 / 字段 |
| JSON | 不显示 | 数据集名称 | JSON数据 | 数据 / 字段 |
| HTTP | 不显示 | 数据集名称、请求地址、请求方法、响应路径 | 请求头、请求体、入参配置 | 数据 / 字段 |
| ES | 不显示 | 数据集名称、ES数据源、查询地址、请求方法、响应路径 | 查询报文、入参配置 | 数据 / 字段 |
| WebSocket | 不显示 | 数据集名称、WebSocket地址 | 消息处理脚本、测试样本 | 数据 / 字段 |
| MQTT | 不显示 | 数据集名称、MQTT数据源、订阅主题、缓存条数、空数据策略 | 测试样本、字段映射表 | 数据 / 字段 |

## Component Boundaries

### `DatasetEditorLayout.vue`

职责：

- 提供统一外层布局。
- 根据是否传入 `metadata` 插槽决定是否显示左侧区域。
- 渲染右上基础设置折叠面板。
- 渲染右下数据预览折叠面板。
- 管理数据/字段 Tab 的视觉结构。
- 提供字段列表插槽，让各编辑器复用现有字段表格内容。

不负责：

- 业务校验。
- 测试接口调用。
- 字段解析。
- 数据集类型判断。
- 具体表单字段生成。

### 各数据集编辑器

职责：

- 保留现有 `formData`、校验规则、测试、解析和保存暴露方法。
- 将普通字段放入两列网格。
- 将大字段和表格配置放入全宽区域。
- 将字段列表表格移动到字段 Tab 插槽。
- SQL、Excel 继续向布局传入表结构插槽。

## Error Handling

本次为前端布局优化，不改变后端接口和错误语义。

各编辑器现有错误处理保持：

- 测试失败仍显示 `ElMessage.error`。
- 字段解析失败仍显示当前错误提示。
- 表结构加载失败仍显示当前错误提示。
- 控制台错误日志保持现状；若后续改动 Java catch 块，需遵守项目异常日志规范。

## Testing

实现后至少运行：

```bash
cd dataRoomFront
npm run type-check
npm run lint
```

人工验证：

- SQL 数据集显示左侧表结构，右侧基础设置和数据预览默认展开。
- Excel 数据集显示左侧表结构，右侧基础设置和数据预览默认展开。
- JSON、HTTP、ES、WebSocket、MQTT 不显示左侧区域，右侧横向占满。
- 基础设置普通字段两列展示，大字段和表格类配置独占整行。
- 点击基础设置标题栏可折叠和展开。
- 点击数据预览标题栏可折叠和展开。
- 数据 Tab 内容与当前预览表保持一致。
- 字段 Tab 内容与当前字段列表保持一致，字段类型和描述仍可编辑。
- 测试、字段解析、入参解析、格式化、保存行为不回退。

## Out of Scope

- 不修改后端数据集接口。
- 不新增数据集类型。
- 不重构为完全配置驱动表单。
- 不持久化折叠状态或 Tab 状态。
- 不改变字段推断、参数解析、测试数据集的业务逻辑。
