# 地图预览区域编码布局设计

## 背景

`/dataRoom/map/index` 页面中，地图预览面板需要展示当前选中地图的区域编码。该信息用于辅助识别当前选中的地图，不是主要标题，也不需要占用地图画布空间。

## 方案

地图预览面板标题栏采用左右布局：

- 左侧保留主标题 `地图预览`。
- 右侧展示 `区域编码：{code}`。
- 仅在当前选中地图存在 `code` 时展示区域编码。
- 区域编码使用 Element Plus 弱文本变量，例如 `var(--el-text-color-secondary)`。
- 区域编码使用常规字重和较小字号，视觉权重低于主标题。
- 标题栏空间不足时，区域编码保持单行并省略，避免挤压主标题或造成换行。

示意：

```text
地图预览                                      区域编码：china
```

## 取舍

该方案将主标题和辅助元信息分开，保留标题栏的信息聚合能力，同时避免把区域编码放入地图画布造成视觉干扰。相比标题下方副标题方案，它不增加标题栏高度；相比画布浮层方案，它不会遮挡地图预览内容。

## 实现范围

仅调整 `dataRoomFront/src/dataroom-packages/map/index.vue` 中地图预览面板标题区域的模板和局部样式。不修改接口、不修改地图渲染逻辑、不新增状态。

## 验证

修改后运行前端类型检查：

```bash
cd dataRoomFront
npm run type-check
```
