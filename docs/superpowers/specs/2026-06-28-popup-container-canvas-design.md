# 弹框与容器画布能力设计草案

**日期**：2026-06-28  
**状态**：草案，仅记录已确认方向，暂不进入实施  
**范围**：大屏设计器 `VisualScreenDesigner` 与页面设计器 `PageDesigner`

## 背景

设计器需要新增弹框能力，并与现有交互动作系统配合使用。用户可以配置多个弹框，弹框支持标题、背景、尺寸、关闭方式、动画等容器样式；弹框内部可以继续添加组件，并且内部组件的样式、数据、交互配置能力应与外层画布一致。

后续还会出现类似轮播的容器类组件：容器内部可以配置多个其他组件。因此本次方案不把弹框设计成孤立能力，而是将“容器内部画布”抽象为可复用能力，弹框、轮播、Tab、面板等容器类组件都可以基于同一套内部画布模型扩展。

## 已确认产品约束

- 弹框、轮播和未来其他容器类组件都应存放在页面 `chartList` 中，保持统一组件树模型。
- 弹框不会在主画布中渲染出来，也不会占据主画布网格位置。
- 顶部导航提供工作区切换：主画布与弹框画布。
- 切换到弹框画布后，从 `chartList` 中过滤弹框类型组件，展示弹框列表。
- 支持新增多个弹框，每个弹框都有自己独立的内部画布。
- 弹框内部组件配置能力与外层画布一致，包括样式、数据、交互配置。
- 弹框内部布局模式在结构上支持扩展，但当前仅实现网格布局模式。
- 弹框运行时只通过交互动作用打开、关闭或切换，不支持默认打开和条件显示。
- 运行时允许多个弹框同时打开，后打开的弹框层级比当前最上层高 `100`。
- 每个弹框可单独配置关闭按钮、遮罩点击关闭、Esc 关闭。
- 弹框内部组件与外层页面完全共享作用域：全局变量、数据集、交互目标池一致。

## 推荐方案

采用 **chartList 统一组件树 + 弹框专属工作区 + 可复用容器画布模型**。

弹框作为一种特殊容器组件进入 `chartList`，例如 `type = 'DrPopup'`。主画布渲染普通组件时过滤掉 `DrPopup`，因此弹框不会出现在主画布，也不会参与主画布布局。弹框工作区从同一个 `chartList` 过滤出 `DrPopup`，生成弹框列表，并为选中的弹框展示独立内部画布。

未来轮播、Tab、分组面板等容器组件也继续作为普通 `ChartConfig` 存放在 `chartList` 中，但它们可以复用与弹框相同的内部画布配置结构。这样既保持组件树统一，又避免为弹框单独引入 `popupList`，减少未来容器能力重复建设。

## 数据模型

### 弹框组件

弹框本身是一个 `ChartConfig<PopupProps>`：

```ts
interface PopupProps {
  title: string
  layoutMode: 'grid'
  style: PopupStyleConfig
  close: PopupCloseConfig
  canvas: ContainerCanvasConfig
}
```

说明：

- `title`：弹框标题。
- `layoutMode`：弹框内部布局模式，首版固定为 `grid`，后续可扩展像素布局。
- `style`：弹框容器样式，包括大小、背景、边框、圆角、阴影、标题栏、关闭按钮位置和动画等。
- `close`：关闭行为配置，包括关闭按钮、遮罩关闭、Esc 关闭。
- `canvas`：弹框内部画布配置。

### 容器内部画布

容器内部画布使用公共结构：

```ts
interface ContainerCanvasConfig {
  layoutMode: 'grid' | 'absolute'
  grid?: {
    colNum: number
    rowHeight: number
    margin: [number, number]
  }
  chartList: ChartConfig<unknown>[]
}
```

首版仅实现 `layoutMode = 'grid'`。`absolute` 只作为结构预留，不在首版 UI 中开放。

### 未来轮播组件复用方式

轮播组件仍然是普通 `ChartConfig<CarouselProps>`，每个轮播页可以持有一个内部画布：

```ts
interface CarouselProps {
  slides: Array<{
    id: string
    title: string
    canvas: ContainerCanvasConfig
  }>
}
```

这保证弹框和轮播共享同一套内部画布编辑能力，但运行时语义仍由各自组件负责。

## 编辑器结构

### 主画布工作区

主画布工作区保持现有设计器体验，只渲染普通组件。渲染前过滤掉弹框类型组件：

```ts
const visibleChartList = chartList.filter((chart) => chart.type !== 'DrPopup')
```

弹框仍然保存在 `chartList` 中，但不显示在主画布，不参与主画布网格布局，也不出现在主画布上的选中、拖拽、缩放流程里。

### 弹框画布工作区

顶部导航增加工作区切换：

- 主画布
- 弹框

切换到弹框工作区后：

- 从 `chartList` 过滤出 `type = 'DrPopup'` 的组件作为弹框列表。
- 支持新增、复制、删除、重命名弹框。
- 选中某个弹框后，中央区域显示该弹框的内部网格画布。
- 内部画布支持添加组件、选择组件、调整布局、配置样式、绑定数据、配置交互。
- 右侧配置面板根据当前选中对象切换：
  - 选中弹框本体时，配置弹框标题、大小、背景、遮罩、关闭行为、动画等。
  - 选中弹框内部组件时，复用现有组件配置面板。

### 组件配置复用

弹框内部组件仍然是标准 `ChartConfig<unknown>`，因此可以复用现有组件自动注册机制：

- `AutoInstall.ts` 获取组件渲染器。
- 组件 `install.ts` 提供默认实例。
- 配置面板继续使用组件自己的 `panel/index.vue`。
- 数据绑定、字段映射、mock 数据、交互行为继续沿用现有模型。

## 交互作用域

弹框内部组件与外层页面共享同一套页面级上下文：

- 全局变量列表共用。
- 数据集选择与运行共用。
- 交互动作目标池共用。
- 弹框内部组件可以控制外层组件。
- 外层组件可以通过交互动作用打开、关闭、切换弹框。

为了保持目标选择体验一致，弹框也作为 `chartList` 中的组件出现在目标选择器中，但需要在 UI 中清晰区分普通组件和弹框组件。
