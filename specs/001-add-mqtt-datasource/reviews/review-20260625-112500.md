## Executive Summary

整体实现质量较高，符合项目现有架构模式（工厂模式、Jackson 多态、流式数据集运行态）。后端类型注册、敏感字段处理、连接测试分类逻辑完整；前端 MQTT 编辑器遵循 Element Plus 默认样式约束，无硬编码颜色或内部样式覆盖。主要关注点集中在：(1) 连接测试端点缺少对 RSA 加密密码的解密处理；(2) `MqttDatasetRuntime` 中 TLS 单向认证配置未集成；(3) `MqttDatasetRuntimeFactory.create()` 缺少空值保护。无安全阻塞项，建议在合并前修复 2 项 Warning。

## Review Findings

| ID | Severity | File | Line(s) | Category | Finding | Recommendation |
|----|----------|------|---------|----------|---------|----------------|
| R001 | 🟡 Warning | MqttDatasourceConnectionService.java | 78-84 | Correctness | 连接测试接口直接使用 `config.getPassword()` 连接 Broker，但前端通过 `encryptByRsa()` 加密后传输密码。测试时收到的是 RSA 密文而非明文，会导致认证必定失败。 | 在 `test()` 方法入口处对 password 字段做 RSA 解密（参考现有 `UserController` 的解密方式），或在连接测试前由 Controller 层统一解密。 |
| R002 | 🟡 Warning | MqttDatasetRuntime.java | 60-73 | Correctness | `start()` 方法仅处理了 `usernamePassword` 认证模式的凭据设置，但缺少 `tlsOneWay` 模式的 SSL/TLS 配置。当数据源配置为 TLS 单向认证时，运行态订阅将以明文连接 Broker，可能导致连接失败或安全降级。 | 复用 `MqttDatasourceConnectionService.buildTrustManagerFactory()` 的逻辑，在 `tlsOneWay` 模式下调用 `builder.sslWithDefaultConfig().sslConfig().trustManagerFactory(...)` 配置 TLS。 |
| R003 | 🟡 Warning | MqttDatasetRuntimeFactory.java | 31-33 | Error Handling | `dataSourceMapper.getByCode()` 可能返回 `null`（数据源已删除），后续 `getDataSource()` 将抛出 NPE。 | 添加 null 检查：若 `dataSourceEntity == null`，抛出 `DataRoomException("引用的MQTT数据源不存在")`。 |
| R004 | 🟡 Warning | MqttDatasourceConnectionService.java | 124-126 | Correctness | `classifyException` 中 `rootMsg.contains("CONNACK") && rootMsg.contains("5")` 缺少括号，`||` 优先级低于 `&&`，导致该条件实际为 `(contains("Bad user name...")) OR (contains("CONNACK") AND contains("5")) OR (contains("NOT_AUTHORIZED"))`，逻辑正确但可读性差且容易在后续修改中出错。 | 为 `CONNACK && 5` 子条件添加显式括号：`|| (rootMsg.contains("CONNACK") && rootMsg.contains("5"))` |

| R009 | 🟢 Suggestion | data-source/components/MqttEditor.vue | 93 | UX | `testConnection()` 在调用前未执行 `validate()`。用户在 host 为空时点击连接测试不会有前端拦截，直接发送请求到后端再由后端校验。 | 建议在 `testConnection()` 开头调用 `await formRef.value?.validate()` 捕获异常提前拦截。 |

## Recommended Actions

### Must Address (Warnings)

1. **R001** — 在连接测试流程中添加 RSA 密码解密，确保用户能使用密码认证模式成功测试连接。
2. **R002** — 在 `MqttDatasetRuntime.start()` 中添加 TLS 配置分支，确保 `tlsOneWay` 模式的数据集运行态能安全连接 Broker。
3. **R003** — 在 `MqttDatasetRuntimeFactory.create()` 中添加数据源不存在的 null 保护。
4. **R004** — 为运算符优先级问题添加显式括号提升可读性。

### Nice to Have (Suggestions)

5. R005/R006 — 清理未使用的 import。
6. R007 — ObjectMapper 改为 static final 共享实例。
7. R008 — 非 MQTT 连接测试的响应语义可后续统一。
8. R009 — 连接测试按钮点击前增加前端表单验证。
